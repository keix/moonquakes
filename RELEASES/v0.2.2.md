# Moonquakes v0.2.2 — Upper Crescent (Sealed)

Moonquakes seals a critical correctness gap in the v0.2 line.

This release is a focused maintenance patch on top of v0.2.1.
It fixes memory-management errors in compiler and closure setup paths that could leak allocations during repeated runtime initialization and test execution.

## Scope of v0.2.2

v0.2.2 extends v0.2.1 by tightening memory ownership and test reliability:

- Fixed `RawProto` cleanup so debug metadata is fully released after compilation
- Fixed GC object-list linkage during closure allocation
- Removed redundant `_ENV` upvalue allocations during closure setup
- Restored clean unit-test execution under repeated runtime initialization
- Aligned TBC unit tests with Lua-level runtime error behavior

This version focuses on memory correctness and test stability.

## Compatibility

Lua 5.4 Test Suite

- The aggregate `passing/all.lua` flow remains passing
- Unit-test execution no longer leaks memory during builtin/runtime initialization
- `zig build test` now completes cleanly once Zig cache permissions are available

This release does not expand compatibility surface.
It makes the validated surface reliable under repetition.

## Critical Fixes

### Compiler / ownership

- Completed `RawProto` deallocation for duplicated upvalue names and local register names
- Removed a cleanup mismatch between parser-owned and pipeline-owned prototype data

### Garbage collector / closure lifecycle

- Fixed closure allocation so child upvalues remain linked in the GC object chain
- Eliminated unnecessary replacement allocations when initializing `_ENV` upvalues
- Prevented startup-time leaks triggered by the built-in Lua `dofile` wrapper and other compiled closures

### Tests

- Restored leak-free repeated test setup using `Runtime.init()` and `VM.init()`
- Updated TBC tests to assert the Lua-visible exception path and message content

## Differences from Lua 5.4

Known Differences

| Behavior               | Lua 5.4      | Moonquakes v0.2.2          |
| ---------------------- | ------------ | -------------------------- |
| `debug.getinfo` fields | Full set     | Partial metadata           |
| `string.dump`          | Lua bytecode | Moonquakes-specific format |
| Weak tables (`__mode`) | Supported    | Not implemented            |

### Structural Differences

- Full debug metadata is still incomplete
- Weak table semantics are not implemented
- `package` library is not implemented

## What v0.2.2 Is Not

The following remain outside the scope of this release:

- ABI stability
- Public C API finalization
- Full debug metadata compatibility and complete hook compatibility
- Weak table semantics (`__mode`)
- Module system (`package`)

## Design Position

v0.2.2 marks the point at which:

- The v0.2 runtime no longer leaks through normal builtin closure setup
- Compiler output ownership is internally consistent
- The test harness can validate behavior without allocator noise masking regressions

This release does not broaden the language surface.  
It hardens the runtime.
