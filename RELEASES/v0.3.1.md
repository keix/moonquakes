# Moonquakes v0.3.1 — First Quarter (Hardened)

Moonquakes hardens the frozen v0.3 line.

This release is a maintenance patch on top of v0.3.0.
It does not widen the Lua-visible compatibility surface.
It restructures core VM subsystems so future optimization work can move faster without loosening the behavioral boundary established in v0.3.0.

## Scope of v0.3.1

v0.3.1 extends v0.3.0 by making internal state transitions explicit and module-scoped:

- Split VM runtime state into clearer internal subsystems
- Encapsulated hook dispatch behind a dedicated high-level event API
- Moved traceback snapshot capture and formatting into a focused traceback module
- Isolated coroutine yield / resume bookkeeping into a dedicated yield-state module
- Introduced explicit error-state lifecycle helpers instead of scattered direct mutation
- Wrapped field-cache coordination behind a smaller internal API
- Removed remaining direct state writes from builtins and GC-adjacent paths

This version focuses on internal correctness, maintainability, and refactor safety.

## Compatibility

Lua 5.4 Compatibility Contract

- The Lua-visible compatibility contract frozen in `v0.3.0` remains the boundary
- `_VERSION == "Lua 5.4"` remains unchanged
- The aggregate suite in `passing/all.lua` remains passing
- `zig build test` passes `357/357` tests
- No new language features or public runtime surface are introduced in this patch

This release is intentionally behavioral-noise-free.
The goal is to make internal evolution safer without changing validated semantics.

## Internal Hardening

### Hook and debug-event flow

- Consolidated call, tail-call, return, line, and count hook dispatch into `src/vm/hook.zig`
- Added higher-level `onX` hook helpers so listener checks, transfer buffers, and re-entrancy handling share one path
- Reduced duplicated hook-state manipulation across opcode execution paths

### Traceback and error flow

- Moved traceback snapshot capture and frame formatting into dedicated VM modules
- Added explicit `ErrorState` ownership for raised values, unwind bookkeeping, close-metamethod state, and native-call depth
- Replaced ad-hoc error-state mutation with lifecycle helpers that make unwind behavior easier to reason about

### Coroutine yield / resume flow

- Isolated suspension metadata and resume result placement into `src/vm/yield.zig`
- Centralized tail-call-aware resume handling instead of spreading it across VM execution code
- Reduced coupling between coroutine builtins and raw VM state layout

### VM structure and cache boundaries

- Grouped field-cache state behind a dedicated module and take/clear-style API
- Continued splitting `VM` internals into smaller subsystem-focused units
- Removed more builtin and GC paths that previously mutated VM internals directly

## Why this release matters

`v0.3.0` froze behavior.
`v0.3.1` strengthens the implementation boundary that protects that behavior.

The immediate benefit is not new Lua functionality.
The benefit is that future optimization and cleanup work can proceed against clearer ownership rules for hooks, traceback capture, yield state, field-cache hints, and error unwinding.

## Differences from Lua 5.4

Known Differences

| Behavior                         | Lua 5.4   | Moonquakes v0.3.1                    |
| -------------------------------- | --------- | ------------------------------------ |
| Weak tables (`__mode`)           | Supported | Not implemented                      |
| Debug metadata completeness      | Full      | Partial compatibility                |
| Traceback formatting             | Full      | Semantic shape tested, not byte-exact |
| `package` edge compatibility     | Full      | Core behavior covered, details remain |
| Public C API                     | Available | Not yet implemented                  |

## What v0.3.1 Is Not

The following remain outside the scope of this release:

- ABI stability
- Public C API finalization
- Weak table semantics (`__mode`)
- Complete debug metadata compatibility
- Full `package` compatibility beyond currently tested behavior
- New Lua-visible language or library features

## Design Position

v0.3.1 marks the point at which:

- The frozen v0.3 compatibility line is backed by clearer internal ownership boundaries
- Hook, traceback, yield, error, and field-cache flows are less entangled with the central VM implementation
- The next round of implementation work can proceed with lower refactor risk

This release does not broaden the contract.
It hardens the machinery behind it.
