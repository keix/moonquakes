# Moonquakes v0.2.1 — Upper Crescent (Locked)

Moonquakes advances the Upper Crescent.

This release is a focused patch on top of v0.2.0.
It closes the remaining gap in the upstream Lua 5.4 test flow by making the integrated `all.lua` execution path run successfully without modifying the upstream-style driver.

## Scope of v0.2.1

v0.2.1 extends v0.2.0 from near-complete validation to full aggregate execution:

- `_VERSION` now reports `Lua 5.4` for compatibility with the upstream suite
- The stand-alone interpreter keeps Moonquakes identity for `-v`
- The integrated `passing/all.lua` orchestration now completes successfully
- Protected-call recovery is fixed in nested `xpcall` error-handler paths
- Branch scopes now close captured locals correctly on `if` / `elseif` / `else`
- The default `make test` flow now runs the aggregate suite directly

This version focuses on integration correctness.

## Compatibility

Lua 5.4 Test Suite

- 100% pass rate in the current `passing/` suite
- Aggregate `all.lua` flow now passes
- `errors.lua` no longer hangs inside protected stack-overflow recovery
- `all.lua` no longer corrupts long-lived locals through unclosed branch scopes

The aggregate compatibility path now runs through successfully end-to-end.

## Integration Fixes

### Runtime compatibility

- Aligned script-visible version semantics with Lua 5.4 by returning `Lua 5.4` through `_VERSION`
- Preserved Moonquakes branding and credits for CLI version output

### Error handling

- Fixed a protected-call recovery bug in nested `xpcall` error handlers
- Restored correct behavior after stack-overflow recovery in `passing/errors.lua`

### Compiler / scope semantics

- Emitted `CLOSE` for `if` / `elseif` / `else` branch scopes when captured locals escape
- Fixed an upvalue lifetime bug that corrupted long-lived locals in the `all.lua` driver

### Tooling

- Updated `make test` to run `passing/all.lua` from inside the `passing/` directory
- Kept colored pass/fail output in the Makefile target

## Differences from Lua 5.4

Known Differences

| Behavior               | Lua 5.4      | Moonquakes v0.2.1          |
| ---------------------- | ------------ | -------------------------- |
| debug.getinfo fields   | Full set     | Partial metadata           |
| string.dump            | Lua bytecode | Moonquakes-specific format |
| Weak tables (`__mode`) | Supported    | Not implemented            |

### Structural Differences

- Full debug metadata is still incomplete
- Weak table semantics are not implemented
- package library is not implemented

## What v0.2.1 Is Not

The following remain outside the scope of this release:

- ABI stability
- Public C API finalization
- Full debug metadata compatibility and complete hook compatibility
- Weak table semantics (`__mode`)
- Module system (`package`)

## Design Position

v0.2.1 marks the point at which:

- Aggregate behavioral validation now runs through end-to-end
- The upstream-style test driver no longer needs local patching
- Protected execution and scope finalization behave consistently under stress

This release does not expand the system surface.  
It closes the integration gap left after v0.2.0.
