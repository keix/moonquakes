# Moonquakes v0.2.0 — Upper Crescent

Moonquakes reaches its Upper Crescent.

This release validates the runtime against the Lua 5.4 test suite and establishes behavioral consistency across the system.
The execution model, memory structure, and runtime semantics are no longer only defined — they are verified.

## Scope of v0.2.0
v0.2.0 extends v0.1.0 from structure to verification:

- Lua 5.4 behavior validated against upstream test suite
- VM execution semantics aligned with reference implementation
- Garbage collector behavior verified under real workloads
- Coroutine lifecycle and yield/resume semantics implemented and tested
- Error propagation and protected call behavior verified across boundaries
- Metamethod dispatch consistency confirmed
- Standard library behavior aligned with Lua 5.4 expectations

This version focuses on behavioral correctness.

## Compatibility

### Lua 5.4 Test Suite

- 94% pass rate (32/34)

| Status  | Test       | Description                          |
| ------- | ---------- | ------------------------------------ |
| pending | all.lua    | full suite orchestration             |
| aligned | heavy.lua  | stress behavior matches upstream Lua |

All core behavioral tests pass.
Remaining tests focus on orchestration and stress validation.

## Execution and Memory Model

The runtime now operates as a closed system:

- VM state is isolated from the host environment
- Memory is explicitly owned and observable
- No external mutation occurs outside defined boundaries
- Execution remains deterministic under controlled input

If the boundary is closed, correctness becomes observable.

## Differences from Lua 5.4

| Behavior                 | Lua 5.4      | Moonquakes v0.2.0          |
| ------------------------ | ------------ | -------------------------- |
| `debug.getinfo` fields   | Full set     | Partial metadata           |
| `string.dump` output     | Lua bytecode | Moonquakes-specific format |

## What v0.2.0 Is Not

The following remain outside the scope of this release:

- ABI stability (not yet frozen)
- Public C API finalization

## Design Position

v0.2.0 marks the point at which:

- Runtime behavior is externally validated
- Execution semantics are stable
- Memory and control flow remain explicit
- The system can be reasoned about as a closed unit

This release does not aim for finality.  
It establishes correctness.
